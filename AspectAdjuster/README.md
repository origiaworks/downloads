# AspectAdjuster for Unity

どんな画面解像度のデバイスでも、指定したアスペクト比（例：16:9）を維持し、必要に応じてレターボックス（上下黒帯）またはピラーボックス（左右黒帯）を自動生成するカメラ制御スクリプトです。

## 主な機能

- **固定アスペクト比の維持**: 画面サイズが変わっても、ゲームの描画領域を指定した比率（16:9、4:3など）に保ちます。
- **レターボックス/ピラーボックス**: 画面が縦に長い場合は上下に、横に長い場合は左右に自動で黒帯を挿入します。
- **エディタリアルタイム反映**: `[ExecuteAlways]` を実装しているため、ゲームを再生しなくても、Gameビューのサイズ変更やインスペクターの値をいじるだけで即座に見た目を確認できます。
- **軽量設計**: 画面サイズに変更があった時のみ再計算を行うため、パフォーマンスへの影響は最小限です。

## セットアップ方法

1.  **ファイルの配置**: `AspectAdjuster.cs` をプロジェクトの `Assets` フォルダ内に配置します。
2.  **アタッチ**: 
    - シーン上の **Main Camera** を選択します。
    - メニューの `Component > Layout > Aspect Adjuster` を選択するか、インスペクターに直接スクリプトをドラッグ＆ドロップします。
3.  **アスペクト比の指定**:
    - `Aspect Vec` に目標とする比率を入力します（例：16:9なら X=16, Y=9）。

## インスペクターの設定項目

| 項目 | 説明 |
|:---|:---|
| **Target Camera** | 調整対象のカメラ。未指定の場合は、このスクリプトがアタッチされているオブジェクトのCameraを自動的に使用します。 |
| **Aspect Vec** | 目標とするアスペクト比。Xが横、Yが縦の比率です。 |

## 注意事項

- **背景の黒帯について**: このスクリプトはカメラの `Viewport Rect` を操作して描画範囲を制限します。黒帯の部分に何も表示したくない場合は、シーン内にカメラが1つだけであることを確認してください（複数のカメラがある場合、背景に別のカメラの映像が映ることがあります）。
- **UI（Canvas）について**: UIを固定アスペクト比の中に収めたい場合は、Canvasコンポーネントの `Render Mode` を `Screen Space - Camera` に設定し、このスクリプトが適用されているカメラを割り当ててください。
